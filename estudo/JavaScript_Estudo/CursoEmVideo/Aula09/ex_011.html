  1. Estrutura DOM :
  2. DOM significa (Document Object Model).
  4. Podemos definir DOM como uma estrutura que ocorre dentro de uma página na Web assim nomeando e estruturando as partes e transformando elas em objetos, e podendo assim definir de uma maneira que JavaScript poderá utilizar para causar interatividade, ou mudar os objetos assim podendo definir

Primeiramente podemos aprender como funciona o console.dir(document.qualquer objeto especifico) uma observação que deve ser feita é que a questão do console.dir é que se somente escrevermos document ele assessará a todas as funcionalidades e objetos presentes dentro do documento, porém se adicionarmos mas alguma coisa após ao ponto inserido no final da palavra documento poderemos ter acesso específico a um objeto de dentro do Pattern(objeto elementar pai na ramificação no nosso caso DOCUMENT) document.


Devemos definir também como podemos chamar os elemnetos de uma página que estarão dentro do documento, para então podermos manipulalos da maneira mais satisfatória possível. Assim podemos utilizar de artificios dentro do JavaScript tais quais:


  1. Por metodologia de tag com o comando getElementsByTagName(`nome da tag`)
  2. Por metodologia de id com o comando getElementById(`ID do elemento`)
  3. Por metodologia de classe com o comando getElementByClass(`nome da classe`)
  4. Feito por metodologia de query, este é o Seletor mais recomendavel:
  5. querySlector("${aqui vai a tag name}${aqui vai o simbolo estilo css}${aqui vai o id ou a classe}")
  6. A metodologia querySlector tem mais uma opção a querySlectorAll(``), na qual o seletor vai pegar todos os objetos/elementos dentro do documento que tenham o nome da tag, e se definido com uma classeeste seletor todas as tags com determinada class(classe) ou ID(indereço).
  7. Podemos dar um exemplo prático:
  8. document.querySelectorAll("div.CenterContent").
  9. Este comando nos retornara até o momento do site a resposta:
  10. NodeList(3)[div.CenterContent,div.CenterContent,div.CenterContent]
  11. E além disso a possibilidade de acessar tudo que está dentro destes três objetos, suas funcionalidades, seus objetos filhos, seus objetos pais e muito mais, possibilitando assim trabalhar de uma maneira mais eficiente com a programação em JavaScript principalmente voltada para o front-end.
